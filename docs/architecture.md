# Architecture Overview

`agent-workspace` is designed around four orthogonal pillars. Each is specified in detail in `SPEC.md`; this document exists to give you the 10-minute mental model.

## 1. Context Layers

Agents do not share one giant project conversation. Context is stratified:

```
┌─────────────────────────────────────────────────────────┐
│ L1  Project Brief         (stable; authored by Owner)   │
├─────────────────────────────────────────────────────────┤
│ L2  Shared Memory         (structured facts, appended   │
│                            on Review APPROVED or by     │
│                            explicit memory.write)       │
├─────────────────────────────────────────────────────────┤
│ L3  Task Packet           (snapshot per Assignment;     │
│                            generated by Lead/Planner)   │
├─────────────────────────────────────────────────────────┤
│ L4  Handoff Package       (structured output per        │
│                            completed Assignment)        │
└─────────────────────────────────────────────────────────┘
```

Key rule: **L3 is generated by the coordinator, not assembled by the worker.** A worker receives only the slice it needs.

## 2. Roles

```
OWNER (human)
  └─► LEAD_AGENT ──┬─► PLANNER_AGENT (optional)
                   │
                   ├─► WORKER_AGENT  (N, may run concurrently)
                   │
                   ├─► REVIEWER_AGENT
                   │
                   ├─► PM_AGENT
                   │
                   └─► INTEGRATOR_AGENT (optional)
```

Every role is `role + capabilities`, not a locked enum. Agents are matched to work by capability tags (`frontend`, `typescript`, `review.code`, `integrator.github`, …).

## 3. Event Bus

Every interesting thing in a project is an immutable row in `ProjectEvent` with a monotonically increasing `seq`. Agents subscribe by cursor:

```
                 ┌──────────────────────────────────────┐
agent polls ────►│  event.list(projectId, sinceSeq=123) │
                 └───────────────┬──────────────────────┘
                                 ▼
              returns ordered slice + new lastSeq (e.g. 187)
```

External systems (GitHub, GitLab, CI) enter via `ExternalEvent` and are **mapped** (not copied) into the internal `ProjectEvent` stream. That way agents only watch one thing.

## 4. Human-in-the-loop via IM Gateway

Anything that needs a human approves through a uniform `Proposal`:

```
 agent action ──► Proposal(PENDING)
                      │
                      ▼
              im-gateway /notify
                      │
   ┌──────────┬───────┼────────┬──────────┐
   ▼          ▼       ▼        ▼          ▼
 email     dingtalk  slack   matrix    telegram
   │          │       │        │          │
   └──────────┴───┬───┴────────┴──────────┘
                  ▼
        user replies "/approve <id>"
                  │
                  ▼
        im-gateway /inbound/<platform>
                  │
                  ▼
        proposal.resolve(APPROVE|REJECT)
                  │
                  ▼
        queued action executes
                  │
                  ▼
         ProjectEvent: PROPOSAL_RESOLVED
```

The gateway is a separate service because IM integration is IO-heavy, platform-specific, and easy to block. The reference gateway in AgentCraft mirrors the shape of `hermes-agent/gateway/` (per-platform adapters + channel directory + delivery retry + pairing flow).

```
                │          Human / Owner                  │
                └──────────┬──────────────────────────────┘
                           │ IM reply
                           ▼
                ┌────────────────────────────────────────┐
                │          im-gateway                     │
                └──────────┬──────────────────────────────┘

    ┌─────────────────────────────────────────────────────┐

       │MCP              │webhook         │MCP
       ▼                 ▼                ▼
    LEAD_AGENT     GitHub / CI       WORKER_AGENTs (N)
                                     REVIEWER_AGENT
                                     PM_AGENT
                                     INTEGRATOR_AGENT
```
